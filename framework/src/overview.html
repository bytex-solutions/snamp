SNAMP is a middleware solution for monitoring of the software components in your enterprise IT-infrastructure. This solution is running on top of OSGi Framework and can be hosted in every OSGi R5.0-compliant containers, such as Apache Felix, Knopflerfish, Eclipse Equinox. It consists of the following parts (bundles):
  • SNAMP Framework - reusable set of classes and interfaces used to write SNAMP components.
  • Resource Connector - provides connection to the managed resource such as server. This is a back-end component of the SNAMP solution.
  • Gateway - provides access to the managed resource in protocol-specific manner (such as SNMP). Gateway provides conversion between management protocol and resource connector interface. This is a front-end of the SNAMP solution.
  • Configuration Manager - provides SNAMP management interface for configuring SNAMP, such as Web console or command-line interface.
For each architectural component the SNAMP solution provides separated Java package:
  • {@link com.bytex.snamp.connector} for resource connectors.
  • {@link com.bytex.snamp.gateway} for management gateways.
  • {@link com.bytex.snamp.configuration} for configuring SNAMP components.
